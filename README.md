<h1>Download by URL</h1>

Простой скрипт, который позволяет из TOML-файла получать Anki-колоду.
Скрипт сделан, для работы с <a href="https://zhukovsd.github.io/java-backend-interview-prep/">методичкой</a> по вопросам Сергея Жукова.

Шаблон вызова:

<pre>download_by_url.exe|download_by_url.py "url" "path"</pre>

url - урл в интернете, указывающий най TOML-файл <br/>
path - путь в вашей файловой системе, куда вы хотите сохранить колоду

Url-ы можно найти в репозитории Сергея Жукова: https://github.com/zhukovsd/java-backend-interview-prep/tree/master/data

После выбора нужной темы, необходимо нажать "Raw", тогда url вида 
"https://github.com/zhukovsd/java-backend-interview-prep/blob/master/data/%D0%90%D1%80%D1%85%D0%B8%D1%82%D0%B5%D0%BA%D1%82%D1%83%D1%80%D0%B0%20%D0%B8%20%D1%80%D0%B0%D0%B7%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D0%BA%D0%B0/Message%20broker.toml" 
превратится в url вида "<a href="raw.githubusercontent.com/zhukovsd/java-backend-interview-prep/refs/heads/master/data/Архитектура и разработка/Message broker.toml">raw.githubusercontent.com/zhukovsd/java-backend-interview-prep/refs/heads/master/data/Архитектура и разработка/Message broker.toml</a>"

Программа работает только с ulr такого вида. 

После создания колоды, если в вопросах были картинки, они окажутся в папке "pictures". В связи с невозможностью программно 
прикреплять картинки к колоде, они сохраняются отдельно. Вам надо скопировать папку, которая находится внутри 
"pictures", в вашу пользовательскую папку Anki для media. У меня на Win 10 она находится в "C:\Users\user\AppData\Roaming
\Anki\ваш пользователь\collection.media"

Картинки можно сохранять в отдельную папку, если указать необязательный параметр --pictures_path "Путь до папки с 
картинками"

<h1>Download by repo</h1>

Скрипт нужен для скачивания сразу всех колод из определённой папки репозитория. Имеет конфигурационный файл config.toml,
в котором прописаны данные о репозитории, месте сохранения колод. 

Параметры конфигурации:

* owner - владелец репозитория, в нашем случае zhukovsd, если будет сделан каккой-то форк, то можно просто будет поменять владельца тут
* repo - название репозитория на github, если репозиторий переименуют, то можно удобно к этому адаптироваться
* sha - это хэш ветки, из которой мы хотим получить данные. В нашем случае достаточно перейти в ветку master и получить хэш самого последнего коммита (нужно для работы github api)
* toml_folder - папка в репозитории, откуда мы хотим скачать колоды Anki
* exclude_tomls - массив toml-файлов, которые мы хотим игнорировать при скачивании
* destination_directory - место сохранения колод Anki в вашей системе

Все параметры можно перееопределить аргументами командной строки, при запуске скрипта. Параметр будет иметь название 
такое же как и в конфигурационном файле. Например: --owner Pupkin --repo Zalupkin.

Так же присутствует опциональный флаг --archive, который позволяет создать архив со всеми колодами, в папке сохранения 
колод.

Т.к. большинство параметров меняться не будет, при стандартном сценарии, то пример вызова скрипта будет таким:

<pre>download_all_by_repo.exe|download_all_by_repo.py --sha "123e22e8c2e73f52e3d6d4dbc527bb14d32a85ea" --destination_directory "Ваш путь для сохранения колод" --archive </pre>
